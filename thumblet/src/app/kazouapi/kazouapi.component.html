❮ Kazou API

Assigning volunteers and vacations in one platform (POC)

De Kazou API is een proof-of-concept platform en API die het boekingssysteem van de vrijwilligers van Kazou vzw schetst. Ook wordt er gebruik gemaakt van login authenticatie door middel van OAuth-based Google Sign-in.

De API die werd volgens de REST-filosofie geschreven. Als technologie voor dit project werd, een backend, geschreven in .NET MVC, met Angular als frontend technologie, gebruikt.

CRUD

Zoals elke REST API beschikt de Kazou API over alle CRUD (Create, Read, Update en Delete) operaties. Ook werd de API beveiligd tegen foutieve invoer zodat deze robuust is. Bij enkele pagina's is het mogelijk om op extra criteria te zoeken, filteren, aantal items per pagina te kiezen, ...

Google login

Gebruikers kunnen met hun Google account inloggen op het platform. Een ingelogde gebruiker kan meer pagina's bezoeken dan een gebruiker die ni√©t ingelogd is. Om aan te tonen dat de Google Auth werkt, wordt de profielfoto en de gebruikersnaam opgehaald. Deze gebruikersnaam en profielfoto worden ook in de header getoond. Ook kan de gebruiker zijn unieke token-ID opvragen om te bekijken als verificatie.

Pagina's die niet bezocht kunnen worden wanneer de gebruiker niet ingelogd is, worden beveiligd door middel van een RouterGuard. Als een gebruiker klaar is met het platform te bezoeken, kan deze uitloggen op de profielpagina.

Externe API

Om de Kazou API extra flair te geven, werd een externe API toegevoegd. Bezoekers kunnen in de pagina 'vakantiebestemmingen' een weerbericht van de locatie in kwestie te zien krijgen. Naast de (gevoels)temperatuur wordt ook de beschrijving van het weer met illustratie getoond.
